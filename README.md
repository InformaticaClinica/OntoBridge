# Ontobridge: A Tool for Normalizing Medical Data Based on Ontologies
## Introduction

Ontobridge is a groundbreaking tool developed with a commitment to open science, utilizing freely accessible software. This tool is built upon the foundation of ontologies, which brings several advantages. The use of ontologies allows for a more structured and semantic approach to data handling, enhancing data interoperability and consistency. This approach also aids in the efficient integration of diverse data sources, making it easier to analyze and derive insights from complex medical data.

The project was conceived and developed by the Informàtica Clínica group at the Hospital Clínic de Barcelona. The core objective of Ontobridge is to transform a local database into a format compatible with potentially any medical standard, thereby facilitating the sharing of data across various platforms and institutions.

The initial version of Ontobridge is tailored for the CDM OMOP (Common Data Model Observational Medical Outcomes Partnership) standard. Efforts are ongoing to extend its capabilities to support other standards, including i2b2 (Informatics for Integrating Biology and the Bedside) and ICGC Argo (International Cancer Genome Consortium Accelerating Research in Genomic Oncology). This versatility highlights Ontobridge's role in advancing medical research and promoting the global exchange of medical information.

## Software Requirements

Ontobridge supports two methods for ontology construction:

1. **Python with Owlready2**: Install Owlready2 using the command `pip install Owlready2`. It allows ontology manipulation in Python. [Owlready2 Documentation](https://owlready2.readthedocs.io/en/v0.42/).

2. **Protégé**: A user-friendly ontology editor. Download from [Protégé website](https://protege.stanford.edu/).

Additional tools for data handling and queries:

- **Ontop**: Converts databases into Virtual RDF Knowledge Graphs. Guide available at [Ontop VKG Guide](https://ontop-vkg.org/guide/).

- **Apache Jena Fuseki**: A SPARQL server for RDF data querying. More information at [Apache Jena Fuseki Documentation](https://jena.apache.org/documentation/fuseki2/).
